Form Validation (Template Driven)

Building upon the basic form, this form adds validation. Every FormControl and every Form maintains the following properties with an accompanying class name between parenthesis. So the classname ng-invalid is added automatically to an inval field or form.

  • valid (ng-valid)/invalid (ng-invalid)
  • pristine (ng-pristine)/dirty (ng-dirty)
  • touched (ng-touched)/untouched (ng-untouched)

Use class names

Use css class names to show valid/invalid files. The css source for example contains: 

.ng-invalid:not(form)  {{ '{' }}
      border-left: 5px solid #a94442;
    {{ '}' }}

This shows all form controls (but not the form itself) with a red left border. Note that the html source, contains two different examples of how to add an error message to a form control that is invalid. The first example uses binding to the [hidden] property. The second example uses *ngIf. Notice that the name and email field work differently due to the fact that one uses the pristine propery, while the other uses touched.

Prevent submit of an invalid form

You can easily prevent a form from being submitted when one of its form controls is invalid. For example like this: 

<button [disabled]="theForm.invalid" type="submit">Submit</button>
