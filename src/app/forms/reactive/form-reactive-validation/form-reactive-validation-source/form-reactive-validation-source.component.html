Form Validation (Reactive)

Building upon the basic form, this form adds validation. Previously you added the required attribute to the input items on your form. Using the Reactive approach, you will have to define this as a Validator, for example:
'username': new FormControl(null, Validators.required)
You can also define an array of Validators like this:
'password': new FormControl(null, [Validators.required, Validators.minLength(6)])
So in general, validation is done using Typescript.

Access a FormControl in your html component

There are two approaches to getting access to your FormControl objects. The first is to reference the form and then get the FormControl like this:
*ngIf="signupForm.get('password').invalid && signupForm.get('password').touched"
The second is to make the FormControl you need to access, accessible from your component. So in your component declare your FormControl like this:
emailFormControl: FormControl;
And the initialize it like this:
this.emailFormControl = new FormControl(null, [Validators.required, this.emailDotComValidator]);
You should now be able to access it in your HTML code like this:
*ngIf="emailFormControl.errors.required"

Custom validator

The component contains a custom validator, that checks that the email address is a dot com address. The method signature looks like this:
emailDotComValidator(formControl: FormControl): {{ '{' }}[key: string]: string{{ '}' }}
It accepts a FormControl and returns a key (string)/value pair (in this case the value is a string as well). A validator should return nothing at all or null when there is no validation error. When validation fails, it shoud return a key value pair, where the key (string) is the error code, and the value may be the error message. Note that the HTML code demonstrates how to show the different error messages.

Dynamic FormControls

The form allows you to enter multiple languages, dynamically. Notice the formArrayName="languages" in the HTML code. It links to: languages': new FormArray([]) in the component. The onAddLanguage method actually adds a new FormControl.
